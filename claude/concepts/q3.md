# Q3 聊天模式

## 定义

Q3 是一种**特殊的单聊模式**，允许单聊双方各自引入一个 AI 智能体（agent）辅助对话。

## 工作方式

- 用户 A 和用户 B 进行单聊
- A 可以绑定自己的 agent，B 也可以绑定自己的 agent
- 每个 agent 有两种工作模式：
  - **托管模式**：agent 自动代替主人发送回复，主人无需介入
  - **草稿模式**：agent 先将生成的回复发给主人审阅，主人确认后才发送
- 在前端界面上，**始终显示为两个用户在对话**，agent 的存在对对方透明（或半透明）
- 底层基于通用 message 结构实现，前端通过特定字段控制显示逻辑

## 命名说明

- **Q3 是产品功能的系列命名**，没有特殊技术含义
- 后续可能出现 Q4、Q5 等，实现原理类似，均基于通用 message + 前端显示控制
- **不改名为 relay 或其他通用词**：Q3 是有意为之的业务代号，应保留

## 相关概念

- **ghost 字段**（`messaging.proto`）：消息的"幽灵发送方"字段，用于实现"代发"效果
  - `draft`：A 创建内容，B 编辑后以 A 的名义发送（草稿审阅场景）
  - `ghostwritten`：B 创建并发送，但显示为 A 发送（托管代发场景）
- Q3 的 agent 代发消息即利用 ghost 字段实现显示层的身份替换
